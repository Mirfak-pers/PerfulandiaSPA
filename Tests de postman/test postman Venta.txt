# Postman Test Cases for VentaController

## 1. Obtener todas las ventas
- **Método:** GET
- **URL:** http://localhost:8080/api/v1/ventas
- **Headers:** Accept: application/json
- **Tests:**
    - Respuesta 200 OK y body es un array de ventas

---

## 2. Obtener una venta por ID (existente)
- **Método:** GET
- **URL:** http://localhost:8080/api/v1/ventas/1
- **Headers:** Accept: application/json
- **Tests:**
    - Respuesta 200 OK y body contiene la venta con id=1

---

## 3. Obtener una venta por ID (no existente)
- **Método:** GET
- **URL:** http://localhost:8080/api/v1/ventas/9999
- **Headers:** Accept: application/json
- **Tests:**
    - Respuesta 404 Not Found o body vacío (según implementación)

---

## 4. Crear una nueva venta
- **Método:** POST
- **URL:** http://localhost:8080/api/v1/ventas
- **Headers:** Content-Type: application/json
- **Body (JSON):**
    ```json
    {
        "fecha": "2024-06-01",
        "total": 15000,
        "detalle": "Venta mostrador"
    }
    ```
- **Tests:**
    - Respuesta 200 OK y body contiene la venta creada

---

## 5. Eliminar una venta existente
- **Método:** DELETE
- **URL:** http://localhost:8080/api/v1/ventas/1
- **Tests:**
    - Respuesta 200 OK o 204 No Content

---

## 6. Eliminar una venta no existente
- **Método:** DELETE
- **URL:** http://localhost:8080/api/v1/ventas/9999
- **Tests:**
    - Respuesta 404 Not Found

---
